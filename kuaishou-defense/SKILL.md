# 快手答辩文档自动填写

快速批量生成快手专利侵权案件的答辩文档。

## 功能说明

根据案件信息自动填充三种法律文档:

1. **授权委托书** - 替换原告、被告、案号、日期
2. **任职证明** - 替换日期
3. **答辩状** - 替换原告、被告、案号、店铺、经营主体、订单号、法院、日期

## 使用方式

### 触发词
- "填写快手答辩文档"
- "生成快手答辩材料"
- "快手案件文档"

### 交互示例

```
帮我填写快手答辩文档:
原告: 蜜蜂跨境科技（深圳）有限公司
被告: 义乌市瑾椿电子商务商行
案号: (2026)京73民初277号
店铺名: 君亿生活百货
订单号: 2507702249209325
法院: 北京知识产权法院
```

### 执行命令

```bash
cd ~/.codeflicker/skills/kuaishou-defense
python3 fill_defense.py \
  --plaintiff "原告名称" \
  --defendant "被告名称" \
  --case-number "案号" \
  --shop-name "店铺名" \
  --business-entity "经营主体" \
  --order-number "订单号" \
  --court "法院名称" \
  --output-dir "~/Downloads"
```

## 模板占位符

所有模板使用 `【占位符】` 格式:

- `【原告】` - 原告名称
- `【被告】` - 被告名称
- `【案号】` - 案号
- `【店铺名称】` - 店铺名
- `【经营主体】` - 经营主体
- `【订单号】` - 订单编号
- `【XXX法院】` - 法院名称
- `【年月日】` - 日期

## 字体格式

- **字体**: 宋体
- **字号**: 小四 (12磅)
- **加粗**: 保持原模板中的加粗格式

## 技术实现

- 使用 `python-docx` 处理 Word 文档
- 整段替换算法解决 runs 分割问题
- 自动设置字体格式并保持原有样式
